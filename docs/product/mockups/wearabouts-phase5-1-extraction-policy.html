Extraction policy for cheaper, faster outfit upload

Wearabouts should detect generously, then generate selectively. The default should make the core outfit closet-ready without spending on every visible accessory.

Problem

The first successful outfit run generated every visible candidate. Quality was strong, but latency and cost were too high because a watch and shoes waited in the same queue as the core garments.

Default policy

Core outfit generation should include the garments that matter most to Closet, Mixer, and Trips. Everything else can be detected and saved for optional generation.

✓ Clear top or outerwearAlways generate the best visible upper-body garment. default
✓ Clear bottomsAlways generate pants, jeans, shorts, or skirt when visible. default
? Prominent shoesGenerate only if visible, high confidence, and large enough. usually
+ AccessoriesDetect, but generate only when the user asks. manual
